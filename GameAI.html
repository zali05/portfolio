Game of the Amazons AI

Overview

The Game of the Amazons AI is a hybrid move‑selection engine designed with a clear separation between game logic, network communication, and UI components. It dynamically chooses between Monte Carlo Tree Search and Minimax with Alpha‑Beta Pruning based on board state, ensuring efficient and accurate decisions throughout the game.

Techniques Employed

Monte Carlo Tree Search (MCTS)

  • When Used: Early-to-mid game when more than 50 cells are empty.
  • How It Works: Builds a search tree via random simulations, using UCT to balance exploration and exploitation.
  • Why Effective: Handles large move spaces without exhaustive search, relying on statistical sampling.

Minimax with Alpha‑Beta Pruning

  • When Used: Late game when fewer than 50 cells are empty.
  • How It Works: Explores moves to a predefined depth, evaluates via heuristic, and prunes branches that cannot improve the outcome.
  • Why Effective: Provides precise, deterministic decisions in tactical endgame scenarios.

Decision Mechanism

The Brain class monitors the number of empty cells and selects MCTS or Minimax accordingly, adapting strategy to the game phase.

Software Engineering Aspects

Modular Design & Separation of Concerns

  • COSC322Test: Interfaces with the game server, GUI, and the Brain class, managing network messages via the GameClient API.
  • Brain: Encapsulates AI logic, including move generation, algorithm selection, and win/loss checks, separate from UI and networking.
  • Move: Represents a queen’s move and arrow placement, supports simulation without altering the live game state.
  • MCTS & Minimax Classes: Implemented independently, allowing easy replacement or tuning of each algorithm.

Maintainability & Extensibility

  • Clear Interfaces: Each class has a well-defined role, minimizing interdependencies.
  • Encapsulation: Complex game logic contained within the Brain class to reduce duplication.
  • API Integration: Communicates with the game server using standardized methods (e.g., sendMoveMessage).
  • Testing & Debugging: Console logs report algorithm choice, move decisions, and state updates for easy troubleshooting.
  • Adaptive Strategy: Dynamic algorithm switching ensures optimal performance across all game phases.

Conclusion

This AI demonstrates a robust hybrid approach, combining MCTS and Minimax with clear modularity, making it both efficient in gameplay and maintainable for future enhancements.
